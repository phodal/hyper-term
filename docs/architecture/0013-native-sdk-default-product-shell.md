# ADR 0013: Adopt Native SDK as the default product shell

- Status: accepted
- Date: 2026-07-18
- Amended: 2026-07-19 (explicit Terminal and Agent tab actions)
- Supersedes the product-shell decision in:
  [ADR 0012](0012-native-sdk-renderer-spike.md)
- Depends on: [ADR 0002](0002-runtime-authority-boundaries.md),
  [ADR 0011](0011-versioned-block-render-document.md), and
  [ADR 0012](0012-native-sdk-renderer-spike.md)

## Context

ADR 0012 kept Tauri as the reference adapter while Native SDK was evaluated.
The evaluation established that Native SDK `v0.5.3` can provide a small native
window host, Metal-backed canvas presentation on macOS, compiled declarative
views, deterministic widget identity, virtualized lists, native scrolling,
system menus, accessibility snapshots, and bounded WebView panes.

It also established a hard limitation: Native SDK has no terminal emulator
widget. Its JavaScript bridge is a bounded JSON control channel, not a PTY or
model-stream data plane. Selecting it as the product shell therefore does not
select a terminal renderer and does not move process authority into Zig.

The product requirement has also become clearer. Hyper Term opens as a normal,
fast Terminal. `New` always opens another ordinary Terminal tab; `Agent` is a
separate, explicit tab action. Agent mode composes ACP, MCP, generated UI,
Computer Use evidence, approvals, editors, and diffs around the terminal
without making a WebView the application shell.

## Decision

Native SDK `v0.5.3` at commit
`57bf56bc58058768d436099c1005bfc66bd55ac3` is the default macOS product shell.
The dependency remains pinned while it is pre-1.0. Tauri is no longer the
default implementation target; it remains a comparison adapter until the
Native path passes the release gates retained from ADR 0012.

```text
Native SDK / Zig desktop host
  window, titlebar, menus, focus, notifications, Block composition
  |
  +-- ordered local control plane ----> Rust hyperd
  +-- ordered terminal plane ---------> Rust terminal model / PTY
  +-- content-addressed artifacts ----> Deno compiler + local store
  +-- bounded WebView Block ----------> trusted editor or isolated GenUI

Rust hyperd
  PTY, zsh lifecycle, transcript, permissions, ACP/MCP, Deno supervision
```

The desktop host is a zero-config Native SDK project. It does not commit a
developer-machine absolute SDK path. CI and release acquisition pin the CLI,
SDK commit, and Zig 0.16 toolchain.

## Session modes

The initial state is always `Terminal`. It exposes ordinary terminal behavior
and minimal session chrome. The session bar exposes two unambiguous actions:

- `New`: direct user-owned zsh session; no AI operation approval is
  required merely to type or run a shell command.
- `Agent`: an explicit Agent tab containing a terminal plus a native Block
  surface for agent messages, plans,
  tools, approvals, diffs, artifacts, and Computer Use evidence.

Sessions use native tab semantics. Each tab has an exact close target and a
context-menu Close action; Command-W closes the active tab. All three paths
enter the same Rust-owned PTY and Agent cleanup lifecycle.

Changing the visible mode never changes operation authority. A model can
propose an action, but only the Rust permission broker can authorize and
dispatch it.

## Rendering planes

The native canvas is the default UI renderer. The terminal plane sends ordered
cell-grid snapshots and damage records, not raw escape sequences interpreted by
Zig and not full-window RGBA frames. A dedicated terminal surface must preserve
selection, cursor, hyperlinks, CJK, emoji, IME, accessibility, and scrollback.

Agent Blocks are semantic data from the Rust `BlockDocument`. Native markup is
application code and is never generated by an agent at runtime.

React, Tiptap, CodeMirror, diff tools, and generated Agentic UI are mounted only
as bounded WebView Blocks. Layout, visibility, origin, focus, bridge capability,
and lifetime are leased by the native compositor. Isolated generated previews
have the bridge disabled. PTY bytes, model streams, source maps, and bundles do
not travel through the WebView bridge.

## Design system

The root `DESIGN.md` is the normative design source using Google's alpha
DESIGN.md format. Native SDK `DesignTokens` and Web CSS variables are adapters
of its semantic values. Native and Web renderers may use different platform
primitives, but they must preserve the same color roles, spacing, typography,
focus, attention, and approval hierarchy.

## Consequences

Hyper Term has one native-first default instead of two equally weighted shell
directions. The React workbench becomes a renderer and editor island, not the
desktop boundary. Rust remains independently testable and can continue serving
daemon, native, WebView, and future clients.

The cost is an explicit Rust/Zig streaming boundary and a purpose-built terminal
surface. Native SDK maturity, cross-platform CPU rendering, packaging, signing,
and update support remain release risks. The replacement gates and benchmark
matrix in ADR 0012 still apply; acceptance of this ADR chooses the direction,
not an exemption from those gates.

## Rejected alternatives

- **Use a full-window WebView as the default UI.** It fails the native-first
  rendering goal and does not isolate generated UI from trusted chrome.
- **Send terminal output through the Native SDK JavaScript bridge.** Its bounded
  request/response semantics are unsuitable for ordered high-volume streams.
- **Let Zig spawn shells or agents directly.** It creates a second process and
  permission authority outside Rust.
- **Generate `.native` markup from ACP or MCP.** It treats untrusted data as
  application code and bypasses the versioned Block renderer contract.
